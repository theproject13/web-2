# Error Handling Pertemuan 17

## Pengertian Error Handling
Error Handling adalah proses mendeteksi, menganalisis, dan menangani kesalahan (error) yang terjadi saat program dijalankan. Tujuannya adalah agar aplikasi tetap berjalan dengan baik serta memudahkan programmer dalam menemukan dan memperbaiki masalah.

## Materi Praktikum

### 1. Cannot Modify Header Information
Terjadi ketika fungsi `header()` dipanggil setelah ada output yang dikirim ke browser.

### 2. Access Denied MySQL
Terjadi ketika koneksi ke database gagal akibat username, password, atau konfigurasi database yang salah.

### 3. Parse Error pada OOP
Terjadi karena kesalahan sintaks, misalnya kurang tanda titik koma (`;`) atau penulisan kode yang tidak sesuai aturan PHP.

## Tujuan Praktikum
- Memahami jenis-jenis error pada PHP.
- Melatih kemampuan debugging.
- Mengetahui cara memperbaiki error yang umum terjadi.
